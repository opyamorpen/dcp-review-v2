# DCP 评审中心 v2 — 需求方案

> 版本：v1.11.33
> 适用范围：ONES 团队级 DCP 决策评审插件
> 文档状态：可交付

---

## 1. 业务背景

半导体行业 IPD（Integrated Product Development，集成产品开发）流程中，DCP（Decision Checkpoint，决策检查点）是关键的阶段性评审决策节点。DCP 评审用于评估项目是否满足进入下一阶段的标准，涉及多角色协同、材料提交、指标量化评估和决议追溯。

传统 DCP 评审依赖线下会议 + 文档传阅，存在以下痛点：
- 材料提交进度不透明，会议前难以确认完整性
- 评审意见分散在邮件/IM 中，难以汇总追溯
- 指标评估缺乏结构化记录，决策依据不清晰
- 决议结果无不可篡改的快照记录，事后争议无据可查
- 关联工作项（风险/缺陷/行动项）手动维护，易遗漏

DCP 评审中心插件在 ONES 项目管理平台上实现 DCP 评审的全流程数字化管理，覆盖模板配置、评审发起、材料提交、多角色评审、决议发布、审计追溯全链路。

---

## 2. 核心业务概念

| 概念 | 说明 |
|------|------|
| **DCP 阶段** | DCP1~DCP4，对应 IPD 流程不同里程碑（概念决策→计划决策→验证决策→量产决策） |
| **评审单** | 一次 DCP 评审的实例，包含项目、阶段、标题、会议时间、状态 |
| **交付物材料** | 评审需要提交的文档/数据，按模板定义，标记必交/选交、负责角色 |
| **关键指标** | 量化评估指标，带黄线/红线阈值和风险颜色（绿/黄/红） |
| **评审人** | 按角色分配的评审参与者，含 Chair（主席）、IPMT 成员等 |
| **决议快照** | 不可覆盖的评审结论记录，冻结全员投票结果，生成唯一快照编号 |
| **审计日志** | 全操作追溯记录，含操作人、动作、目标、结果、时间戳 |
| **Checklist** | 评审检查清单，按角色匹配，评审人逐项勾选 |

## 3. 用户角色与权限

系统定义三个权限点，对应三类用户角色：

### 3.1 DCP 插件管理员（dcp_admin）

- **典型用户**：流程管理员、PMO
- **可执行操作**：
  - 配置 DCP 阶段模板（DCP1~DCP4 名称、决议选项、决议模板）
  - 配置材料模板（材料名称、必交标记、负责角色、适用阶段）
  - 配置指标模板（指标名称、单位、阈值类型、黄/红阈值）
  - 配置评审角色（角色名称、是否必投、是否一票否决）
  - 配置 Checklist 模板（检查项文本、适用阶段、负责角色）
  - 配置通知规则（触发点、通知渠道）

### 3.2 评审发起人（dcp_create_review）

- **典型用户**：项目经理、产品经理
- **可执行操作**：
  - 在项目内创建 DCP 评审单
  - 编辑评审材料（上传/下载/预览/清除附件）
  - 编辑关键指标值
  - 配置评审人列表
  - 发起评审（草稿→评审中）
  - 删除评审单（仅草稿状态、仅创建者）
  - 查看评审详情全部 Tab

### 3.3 评审总览查看者（dcp_view_review）

- **典型用户**：部门负责人、PMO、所有相关方
- **可执行操作**：
  - 查看团队全部 DCP 评审统计面板
  - 按项目分组浏览评审列表
  - 点击评审单查看完整详情（与项目内一致）
  - 复制评审编号和直链分享

> 注：评审人（被分配到评审单的用户）无需额外权限点，通过评审人工作台（dcp-reviewer-workspace）访问待办评审。

## 4. 功能需求

### 4.1 模板配置（dcp_admin）

管理员通过侧边栏「DCP评审中心 → 模板配置」进入全局模板配置页面，维护以下六类配置：

#### 4.1.1 阶段模板管理

- 新增/编辑/删除 DCP 阶段（如 DCP1 概念决策、DCP2 计划决策）
- 每个阶段配置：阶段代码、阶段名称、决议选项（JSON 数组）、决议模板文本、排序
- 决议选项示例：`["通过", "有条件通过", "不通过"]`
- 决议模板：预填充决议正文模板，发布决议时作为初始内容

#### 4.1.2 材料模板管理

- 新增/编辑/删除材料模板
- 每个材料配置：材料名称、适用阶段（多选）、是否必交、负责角色、排序
- 创建评审单时，系统自动按所选阶段带出适用材料模板

#### 4.1.3 指标模板管理

- 新增/编辑/删除指标模板
- 每个指标配置：指标名称、适用阶段、单位、阈值类型、黄线阈值、红线阈值、排序
- 阈值类型：达标即通过 / 高于阈值预警 / 低于阈值预警
- 系统根据当前值和阈值自动计算风险颜色（绿/黄/红）
- 发起评审时校验：存在红线指标的，不允许发起

#### 4.1.4 评审角色管理

- 新增/编辑/删除评审角色
- 每个角色配置：角色名称、是否必须投票（must_vote）、是否有一票否决权（has_veto）、排序
- 系统预置 6 个默认角色：Chair、研发VP、市场VP、财务代表、质量代表、供应链代表
- 发起评审时校验：所有 must_vote 角色必须已指定评审人

#### 4.1.5 Checklist 模板管理

- 新增/编辑/删除检查项
- 每个检查项配置：检查项文本、适用阶段、负责角色
- 评审中状态时，评审人按角色匹配逐项勾选

#### 4.1.6 通知设置

- 通知启用开关
- 三个触发点：发起评审时、全部评审人提交时、发布决议时
- 五种通知渠道：邮件(Email)、微信(WeChat)、钉钉(DingDing)、飞书(Lark)、有道(YouDu)

### 4.2 项目内评审工作台（dcp_create_review）

在项目页面内以页签形式展示，是创建和管理评审单的主入口。

#### 4.2.1 评审列表

- 分页展示当前项目的 DCP 评审单（每页 5~15 条，按视口高度自适应）
- 列：编号、阶段、标题、状态、资料进度、评审人进度、工作项数、会议时间
- 编号蓝色等宽字体，可点击复制编号+直链
- 标题可点击进入详情
- 顶部「+ 发起评审」按钮（需 dcp_create_review 权限）
- 「🔄 刷新」按钮

#### 4.2.2 创建评审单

- 选择 DCP 阶段（从阶段模板加载）
- 填写评审标题
- 选择计划会议时间
- 系统自动按阶段带出适用材料模板和指标模板
- 创建后进入草稿状态，自动生成评审编号（格式：`{项目标识}{YYYYMMDD}{两位序号}`）

#### 4.2.3 评审详情（5 个 Tab）

**Tab 1：📋 评审资料**

- 交付物清单表格：材料名称、责任角色、附件
- 必交材料用红色 `*` 前缀标记（不设单独列）
- 进度格式：「已上传 X/Y」
- 草稿状态可编辑：选择文件（蓝色按钮，底色 #1677ff）、下载、清除
- 附件 📎 文件名可点击预览，下载按钮
- Office 文件不支持 iframe 预览，显示提示而非触发下载
- 下载用隐藏 `<a download>` 当前窗口触发，不开新标签
- 附件列宽度 360px

**Tab 2：👥 评审人与意见**

- 评审人表格：角色、评审人（显示姓名非 UUID）、结论、风险、意见、状态
- 草稿状态可编辑评审人列表（添加/删除评审人、分配角色）
- 评审中状态可查看各评审人提交的意见

**Tab 3：🔗 关联工作项**

- 展示评审单关联的 ONES 工作项列表
- 按创建者汇总显示
- 可关联已有工作项、新建工作项

**Tab 4：📄 决议快照**

- 评审中状态 + Chair 角色可生成决议
- 决议结果：通过 / 有条件通过 / 不通过
- 发布后冻结全员投票快照，生成唯一快照编号
- 决议不可覆盖

**Tab 5：📝 审计日志**

- 全操作记录，按时间倒序
- 字段：时间、动作（中文）、详情、结果
- 展示前 20 条

#### 4.2.4 评审单操作

- 草稿状态：编辑材料/指标/评审人 → 发起评审（三重校验：必投角色已指定、必填材料已上传、关键指标无红线）
- 评审中状态：等待评审人提交意见 → Chair 生成决议 → 发布
- 已完成/已否决：只读
- 删除：仅草稿状态、仅创建者可操作

### 4.3 评审人工作台

评审人通过顶部导航「DCP评审工作台」进入个人工作台。

#### 4.3.1 评审列表

- 待评审 / 已评审 Tab 切换
- 待评审列表：编号、项目、阶段、标题、状态、我的角色
- 已评审列表：额外显示我的结论（✅/⚠️/❌）
- 支持深链接直达（`?review_uuid=xxx`）

#### 4.3.2 评审详情（评审人视角）

- **评审资料（只读）**：材料名称、责任角色、附件（📎 预览+下载）
- **关键指标（只读）**：指标、单位、黄/红阈值、当前值、风险（绿/黄/红背景色）
- **我的评审意见**：
  - 结论（必填，不填充默认值，需手动选择）
  - 风险等级（低/中/高）
  - 意见摘要（结论为「有条件通过」或「不通过」时必填）
- **其他评审人意见**：查看已提交的评审意见
- **Checklist**：按角色匹配的检查项，逐项勾选
- **关联工作项**：关联/新建工作项

### 4.4 团队评审总览（dcp_view_review）

通过侧边栏「DCP评审中心 → 评审总览」进入团队级全景视图。

#### 4.4.1 统计面板

四个统计卡片：
- 全部评审（蓝色）
- 评审中（黄色）
- 已完成（绿色）
- 关联工作项（紫色）

#### 4.4.2 评审列表

- 按项目分组展示（可折叠/展开，默认折叠）
- 每组显示项目名称和评审单数量
- 组内列表列：编号、阶段、标题、状态、资料、评审人、工作项、会议时间
- 支持按阶段、标题、会议时间排序
- 点击行打开评审详情

#### 4.4.3 评审详情

- 复用项目内评审工作台的 `ReviewDetail` 组件，与项目内打开评审单完全一致
- 5 个 Tab（评审资料/评审人与意见/关联工作项/决议快照/审计日志）全部可用
- 顶部「← 返回总览」按钮回到列表

#### 4.4.4 深链接直达

- 支持 URL 参数 `?review_uuid=xxx` 直接打开指定评审单
- 直链场景额外显示「← 返回 ONES 主界面」按钮
- 评审编号可复制（编号+链接到剪贴板）

## 5. 评审流程

### 5.1 状态流转

```
┌─────────┐     发起评审      ┌──────────┐   Chair发布决议   ┌──────────┐
│  草稿    │ ──────────────→ │  评审中   │ ──────────────→ │ 已完成   │
│ (draft) │  三重校验通过     │(reviewing)│  全员提交后      │(completed)│
└─────────┘                  └──────────┘                  └──────────┘
     │                            │                              │
     │ 创建者可删除                │ Chair可发布否决决议            │
     │ 可编辑全部内容              │ 评审人提交意见                 │ └→ 已否决(rejected)
     ▼                            ▼
```

### 5.2 各状态可执行操作

| 状态 | 发起人 | 评审人 | Chair |
|------|--------|--------|-------|
| **草稿** | 编辑材料/指标/评审人、发起评审、删除 | — | — |
| **评审中** | 查看详情 | 提交评审意见、勾选Checklist、关联工作项 | 生成/发布决议 |
| **已完成** | 只读 | 只读 | 只读 |
| **已否决** | 只读 | 只读 | 只读 |

### 5.3 发起评审校验（三重）

1. **角色校验**：所有 `must_vote=true` 的角色必须已指定评审人
2. **材料校验**：所有 `required=true` 的材料必须已上传附件
3. **指标校验**：关键指标不能存在红线（risk_color=red）

### 5.4 决议发布规则

- 仅 Chair 角色可操作
- 所有 `must_vote=true` 角色必须已提交意见
- 通过决议需 ≥3 位非 Chair 评审人投「通过」或「有条件通过」
- 发布后冻结全员投票快照到 `based_on_votes` 字段
- 生成唯一快照编号 `DCP-RES-{时间戳36进制}`
- 决议不可覆盖，不可重新发布

## 6. UI 设计规范

以下规范从实际代码中提取，为前端实现的强制约束：

| 编号 | 规范 | 说明 |
|------|------|------|
| 1 | 评审人显示姓名非 UUID | 通过团队成员 API 解析 UUID→姓名 |
| 2 | 材料必交用红色 `*` 前缀标记 | 不设单独「必交」列，必交材料名称前加红色星号 |
| 3 | 进度格式「已上传 X/Y」 | 标题处显示，如「交付物清单（已上传 3/5）」 |
| 4 | 附件 📎 文件名可点击预览 | 文件名加下划线，点击触发预览 |
| 5 | 下载按钮独立 | 下载按钮紧邻文件名，绿色边框 |
| 6 | 清除按钮纯文字 | 清除按钮只保留「清除」文字，不加垃圾篓图标 |
| 7 | risk_level 中文显示 | 低/中/高，非 low/medium/high |
| 8 | 审计日志 action 用中文 | 如「创建评审」「上传材料」「提交评审意见」 |
| 9 | review_number 蓝色等宽字体 | `fontFamily: 'monospace', color: '#1677ff'` |
| 10 | 下载用隐藏 `<a download>` | 当前窗口触发，不开新标签页 |
| 11 | Office 文件不支持 iframe 预览 | 显示提示文字而非触发下载 |
| 12 | 选择文件按钮底色 #1677ff | 与「发起评审」按钮一致蓝色底白字 |
| 13 | 附件列宽度 360px | 避免上传附件后换行 |
| 14 | 评审结论不填充默认值 | 评审人提交意见时，结论下拉框默认空，需手动选择 |
| 15 | 状态标签圆角小色块 | `background: {color}1a, color: {color}, borderRadius: 4` |
| 16 | 统计卡片大数字 | `fontSize: 28, fontWeight: 700`，下方小标签 |

## 7. 非功能需求

### 7.1 数据存储

- 文件附件上限 10MB（base64 内嵌存储于实体 `file_data` 字段）
- 文本字段上限 ~64KB（text 类型，最大 length 配置）
- 实体存储使用 cursor 分页查询（每页 200 条，安全上限 100 页）
- 父实体 JSON 快照兜底（reviewers_json、checklist_json），防子实体查询丢失

### 7.2 审计与合规

- 全操作审计日志追溯，记录操作人 UUID、动作、目标、详情、结果、时间戳
- operator_uuid 由前端传入（后端无法获取当前用户）
- 决议快照不可覆盖，冻结全员投票结果

### 7.3 权限安全

- 三级权限控制：dcp_admin / dcp_create_review / dcp_view_review
- 前端权限验证通过 batch_check API（需 `Ones-Plugin-Id: built_in_apis` 头）
- 评审人通过评审人工作台访问，无需权限点分配

### 7.4 通知

- 非阻塞通知，通知失败不影响业务流程
- 三个触发点：发起评审、全部评审人提交、发布决议
- 五种渠道：邮件、微信、钉钉、飞书、有道

### 7.5 性能

- 评审列表分页，每页 5~15 条按视口高度自适应
- 团队成员列表缓存（避免重复请求）
- 项目名称前端兜底补查（后端无法调页面 API）
